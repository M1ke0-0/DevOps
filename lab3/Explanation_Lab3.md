## 1. Что такое Docker и зачем он нужен?

**Docker** — это инструмент, который позволяет запустить приложение в изолированной среде, называемой **контейнером**. 
- **Основная проблема:** "У меня на компьютере всё работает, а на сервере — нет".
- **Решение Docker:** Мы упаковываем приложение вместе со всеми его зависимостями (библиотеками, настройками, системными файлами) в один **образ (image)**. Этот образ гарантированно запустится везде, где установлен Docker.

### Ключевые понятия:
1.  **Образ (Image)**: "Чертеж" или неизменяемый шаблон. Содержит ОС (обычно легкий Linux), среду выполнения (Python, Java, Node.js) и твой код.
2.  **Контейнер (Container)**: Запущенный экземпляр образа. Это "живой" процесс, который можно остановить, удалить или перезапустить.
3.  **Dockerfile**: Текстовый файл с инструкциями по сборке образа.

## 2. Часть 1: Кастомизация Nginx

**Nginx** — это популярный веб-сервер. По умолчанию он показывает стандартную страницу "Welcome to nginx!".

### Файлы:
- `index.html`: Твоя персонализированная страница, где написано "My name is Kiril".
- `Dockerfile.nginx`: Инструкция для создания своего образа на базе Nginx.

### Что делает Dockerfile.nginx:
```dockerfile
FROM nginx:latest                      # Берем официальный образ nginx последней версии
COPY index.html /usr/share/nginx/html/index.html # Копируем НАШ файл ВНУТРЬ образа, заменяя стандартный
```

### Команды запуска:
1.  `docker build -t nginx-custom -f Dockerfile.nginx .` — Собираем образ и называем его `nginx-custom`.
2.  `docker run -d --name nginx-lab3 -p 8000:80 nginx-custom`
    - `-d` — запустить в фоновом режиме (detached).
    - `--name nginx-lab3` — даем контейнеру имя.
    - `-p 8000:80` — **проброс портов**. Мы "связываем" порт 8000 нашего компьютера с портом 80 внутри контейнера.

## 3. Часть 2: Python (Flask) приложение

Здесь мы учимся упаковывать реальное приложение на Python.

### Файлы:
- `python/app.py`: Код сервера на Flask.
- `python/requirements.txt`: Список библиотек (в данном случае `flask`).
- `python/Dockerfile`: Инструкция сборки.

### Как работает Dockerfile:
1.  `FROM python:3.12-slim`: Используем легкую версию Python 3.12.
2.  `WORKDIR /app`: Создаем папку `/app` внутри контейнера и переходим в неё.
3.  `RUN pip install -r requirements.txt`: Устанавливаем зависимости.
4.  `CMD ["python", "app.py"]`: Указываем команду, которая выполнится ПРИ ЗАПУСКЕ контейнера.

## 4. Часть 3: Java приложение

Аналогично Python, но с компиляцией.

### Файлы:
- `java/Main.java`: Код сервера на Java.
- `java/Dockerfile`: Инструкция сборки.

### Особенность:
В Java-контейнере нам нужно сначала **скомпилировать** код (`javac Main.java`), а потом запустить скомпилированный класс (`java Main`). Это и прописано в Dockerfile.

## 5. Полезные команды для понимания состояния

1.  `docker ps`: Показывает все запущенные контейнеры. Ты увидишь `nginx-lab3`, `python-lab3` и `java-lab3`.
2.  `docker logs <имя>`: Позволяет посмотреть "консоль" запущенного приложения. Если ты зашел на сайт через браузер, в логах `nginx-lab3` появится запись о GET-запросе.
3.  `docker inspect <имя>`: Выдает гигантский JSON-файл со всеми деталями: какой IP у контейнера, какие порты проброшены, где лежат файлы.
4.  `docker stop <имя>`: Останавливает контейнер.
5.  `docker rm <имя>`: Удаляет контейнер (но не образ).
